成员：杜必林，邝光前，徐志文，欧阳学海，陈松，罗石发

一：任务安排

主页面 （徐志文）

主要难度：

(1).轮播图（包括拖拽功能）

(2).商品链接详情页，查看更多链接分类页（通过href传值跳转）

(3).搜索框

分类页面（含两页面） （杜必林）

主要难度：

(1).分类页面只能一个，数据分类有难度

(2).链接详情页(通过href传值跳转至详情页)

登录注册页面（四个页面） （邝光前）

(1).页面太多

(2).登录注册可以用cookie或者localstorage,或者sessionstroage写，数据存在json

购物车页面,详情页联动 （欧阳学海）

主要难度

(1)购物车数据和后台交互，单独一个json放购物车数据，只有登录才能加入购物车

(2)购物车页面实现增删改查

(3)详情页面只能一个，动态获取数据，接收href传来的值

茶饮机页面,活动页面 （罗石发）

主要难度

(1).活动页面可以参照分类页面

(2).实现动态切换

个人信息页面,活动页面 （陈松）

主要难度

(1).显示个人信息

(2).与登录注册实时同步

代码整合

二、相关规范

移动端兼容及布局规范

采用弹性盒子布局，尽量脱离固定样式布局

实现各种移动端设备的适配兼容

页面命名规范

登录页面：login.html

注册页面：register.html

我的页面：my.html

个人信息页面：info.html

主页面：index.html

分类页面：sort.html

详情页面：detail.html

活动页面：activity.html

茶饮机页面：machine.html

**如需要扩展页面，必须提前通知，商讨之后动工

gulp文件夹命名及层级规范

工程文件名hecha

1.build文件夹{
    1_1  images文件夹:存放图片
    1_2  fonts文件夹：存放字体
    1_3  prd文件夹：{
        scripts文件夹{
            app.index.js
            app.login.js.......
        }
        styles文件夹{
            app.index.css
            app.login.css.......
        }
    }
    1_4  ver文件夹{版本管理}
    1_5  html页面：index.html,login.html等等
}
2.images文件夹{
    （为了方便代码管理，页面对应文件夹）
    2_1  index.images文件夹{index页面的图片}
    2_2  login.images文件夹{login页面的图片}
    2_....（如果一个人写多个页面，可以只创建一个文件夹）
}
3.fonts文件夹{
    iconfont-menu.ttf
    iconfont-menu.woff
    （命名的时候一定要语义化）
}
4.mock文件夹{
    a.json
    b.json
    c.json
    ....(json文件到时再议)
}
5.node_modules文件夹{
    存放模块工具
}
6.src文件夹{
    styles文件夹{
        libs文件夹{提供，切勿乱修改}
        usage文件夹{
            common文件夹{存放各页面之间相同的css部分，主要是    footer部分}
            core文件夹{勿动}
            element文件夹{勿动}
            export文件夹{勿动}
            fragment文件夹{勿动}
            layout文件夹{勿动}
            module文件夹{
                （主要存放每个页面各自的样式）
                m.index.scss
                m.login.scss
                ....
            }
            page文件夹{
                （各页面样式的出口）    
                app.index.scss
                app.iogin.scss
                ....
            }

        }
    }
    scripts文件夹{
        libs文件夹：{主要存放框架等第三方组件}
        tpls文件夹：{
            （主要存放各页面的string文件，公共部分要提取出来，比如说footer.string）
            index.string
            login.string
            ....
        }
        utils文件夹{
            （存放公共js文件，比如购物车显示数量的js）
            common.util.js
            ....
        }
        views文件夹{
            （存放各页面单独的js）
            index.js
            login.js
            ....
        }
        各js出口文件，比如
        app.index.js
        app.login.js
        ...
        ...
    }
}
7.gulpfile.js文件
8.html文件...（以index.html为例）{
    <!DOCTYPE html>
        <html lang="en">

        <head>
            <meta charset="UTF-8">
            <meta content="width=device-width,initial-scale=1.0,maximum-scale=1.0,user-scalable=no" name="viewport">
            <meta content="yes" name="apple-mobile-web-app-capable">
            <meta content="black" name="apple-mobile-web-app-status-bar-style">
            <meta content="telephone=no" name="format-detection">
            <meta content="email=no" name="format-detection">
            <meta http-equiv="X-UA-Compatible" content="ie=edge">
            <title>footballSNS</title>
            <link rel="stylesheet" href="/build/prd/styles/swiper-3.3.1.min.css">
            <link rel="stylesheet" href="/build/prd/styles/app.css">
        </head>

        <body>
            <script src="/build/prd/scripts/zepto.js"></script>
            <script src="/build/prd/scripts/swiper-3.3.1.jquery.min.js"></script>

            <script src="/build/prd/scripts/template.js"></script>
            <script src="/build/prd/scripts/app.js"></script>
        </body>

        </html>
    }
9.package.json文件{安装组件的时候记得要本地安装 加-D}

js和scss规范

必须用scss，别用css，公共模块一定要单独提取出来，比如主页面的footer部分

js尽量封装，用面向对象思想,做到一个函数，任何地方都能引用

js各模块关系明确，不能出现循环依赖
